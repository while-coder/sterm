# SSH 与会话

## 认证方式

STerm 支持密码、私钥、SSH Agent 和交互式认证。交互式认证会显示服务器提出的问题，适合 OTP 验证码及其他挑战应答流程。

私钥路径支持 `~`。Windows 使用 SSH Agent 时会优先连接 OpenSSH Agent，不可用时再尝试 Pageant。

## 主机指纹

STerm 使用自己的 Known Hosts 文件。首次连接必须确认指纹后才能继续；已保存服务器的密钥发生变化时，会同时展示旧指纹与新指纹，并要求明确确认替换。

意外的指纹变化可能意味着服务器正常换钥，也可能是中间人攻击。继续之前应通过其他可信渠道向管理员核对。

## 跳板机与代理

在连接表单的「连接与重连」中，可以选择另一条已保存连接作为跳板机。STerm 会先登录跳板机，再通过它建立目标 SSH 连接。

初始连接还可以经过：

- HTTP CONNECT 代理
- SOCKS5 代理
- 可选的代理用户名和密码

同时配置跳板机与代理时，代理用于连接跳板机。

## 端口转发

一条连接可以自动启动多条转发规则：

- **本地转发：** 在当前设备监听，通过 SSH 访问远端目标。
- **远程转发：** 在 SSH 服务器监听，把流量转发到当前设备能够访问的目标。
- **动态 SOCKS5：** 在本机提供一个由 SSH 隧道承载的 SOCKS5 代理。

会话关闭时监听器会一并停止。除非确实需要给其他设备使用，否则不要把监听地址设为 `0.0.0.0`。

## 连接可靠性

每台机器可以单独设置连接超时和 Keepalive。已成功连接的会话意外断开后，会按逐渐增加的间隔自动重连，最长等待 30 秒。标签页不会被关闭，也可以从状态栏立即手动重连。

## 终端效率工具

- 按 `Ctrl+F` 或点击「搜索」查找终端缓冲区内容。
- 点击「导出日志」保存当前滚动记录。
- 多行粘贴确认可拦住来源不明的一组命令。
- 分屏可以并排查看两个实时会话。
- 广播会把键盘输入发送给所有已连接标签。
- 常用命令可以从状态栏的「命令」菜单执行。
- 每台机器可设置登录脚本，在每次连接成功后准备目录或环境。
