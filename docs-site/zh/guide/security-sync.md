# 安全与同步

## 本地保护

机器列表使用主密码加密。主密码和 AI API Key 通过系统凭证库保存；连接密码、私钥口令和代理密码只有在明确勾选记住后才会进入加密数据。

修改主密码时会重新加密本地机器列表。请妥善保存新密码，STerm 无法在不知道主密码的情况下恢复这些数据。

## 服务器身份

服务器指纹单独保存在 STerm 的 Known Hosts 文件中。首次连接和密钥变化都会要求确认。指纹发生变化时，应先向服务器管理员核对，再选择替换。

## 配置同步

配置同步是 Free 版功能，可选择以下任一方式保存加密载荷：

- GitHub Gist
- Gitee Gist
- GitLab Snippet（GitLab.com 或自建实例）
- 坚果云 WebDAV

同步内容可以包含：

- 已保存连接及删除记录
- AI 模型设置和常用命令
- 加密后的 AI API Key
- 需要在其他设备恢复的私钥内容

面板布局、本机快捷键、缓存目录等设备偏好不会参与同步。

在「设置 → 同步与安全」中选择同步方式并按提示配置凭证。GitHub、Gitee 和 GitLab 会自动复用已有的 STerm 代码片段，找不到时自动创建；GitLab 同时支持 GitLab.com 和自建实例；坚果云会把密文保存到 `STerm/connections.enc`。所有方式都需要在其他设备使用同一主密码解密。

## 导入与导出

机器列表也可以导出为 STerm 加密文件，再在另一台设备导入。请留意导入告警：无法读取或恢复的私钥，可能需要在目标设备上重新选择。

不要在公开 Issue 中发送导出文件、主密码、同步令牌、WebDAV 应用密码或 License Key。
